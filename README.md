# Eye Tracking en Entornos Web

## Objetivo
Aplicar técnicas de eye-tracking para analizar el comportamiento visual
de los usuarios y evaluar la usabilidad de una interfaz web mediante
WebGazer.js y Heatmap.js.

## Descripción de la Página
Se diseñó una página de cursos online con:
- Encabezado con título.
- Bloque de contenido principal.
- Botón CTA “Registrarse Ahora”.

El objetivo fue evaluar si el usuario detecta rápidamente el botón de registro.

## Tarea del Usuario
“Encuentra y haz clic en el botón de registro”.

## Solución Implementada
- **WebGazer.js** para capturar coordenadas de la mirada.
- **Heatmap.js** para visualizar zonas de atención.
- Registro de 45 segundos de interacción.

## Resultados Observados
- Mayor concentración en el bloque de contenido.
- Atención moderada en el encabezado.
- Atención tardía en el botón CTA.

## Análisis de Usabilidad
**Visibilidad del estado del sistema:**  
El botón no resaltaba suficientemente.

**Reconocimiento antes que recuerdo:**  
El usuario debía buscar activamente el CTA.

**Diseño minimalista:**  
La interfaz es simple, pero el contraste del CTA era bajo.

## Mejoras Propuestas
1. Aumentar contraste y tamaño del botón CTA.
2. Ubicar el CTA más cerca del encabezado.
3. Añadir icono visual o animación sutil.
4. Reducir texto del bloque principal.

## Conclusión
El eye-tracking permitió identificar que el CTA no captaba atención inmediata,
evidenciando oportunidades claras de mejora en jerarquía visual.
